This site stays **static-only by design**, so there is no backend contact form, inbox API, or account system attached to it.

The cleanest ways to get in touch are:

- **Email** for direct project or opportunity conversations
- **GitHub** for code, issues, and implementation context
- **LinkedIn** for professional outreach

The email address in this site is already tied to the current git profile, and additional public links can be added in `src/lib/site.ts` whenever they are ready.
